kettle

1.简介

  • 国外开源的ETL工具，纯Java编写
  • 允许你管理来自不同数据库的数据，通过提供一个图形话的用户环境来描述你想做什么
  • 有两种脚本文件，transformation和job，transformation完成针对数据的基础转换，job则完成对整个工作流的控制

1.1.1 结构

1.spoon

​ spoon是构建ETL jobs和transformations的工具。spoon以拖拽的方式图形化设计，能够通过spoon调用专用数据集成引擎或者集群

​ Data Integration Server是一个专用的ETL Server

2.Enterprise Console 企业控制台

​ 提供了一个小型客户端，用于管理Pentaho Data Integration 企业版的部署

1.1.2 核心组件

名称 描述
spoon 图形接口，用于编辑作业和转换桌面应用
Pan 一个独立的命令行程序，用与执行由spoon编辑的转换和作业
Kitchen 一个独立的命令行程序，用于执行由spoon编辑的作业
Carte 一个轻量级的web容器，建立远程的ETL Server

2.2 转换

  • 包括一个或多个步骤（step），如读取文件、过滤数据行、数据清洗或将数据加载到数据库
  • 转换里的步骤通过 跳（hop）连接，跳定义一个单项通道，允许数据从一个步骤向另一步骤流动
  • 在kettle里，数据的单位是行，数据流就是数据行从一个步骤到另一个步骤的移动

2.3 步骤

步骤是转换里的基本组成部分

步骤的关键特性：

  • 需要一个名字，在转换范围内唯一
  • 每个步骤都会读、写数据行
  • 步骤将数据写到与之相连的一个或多个数据跳，再传输到跳的另一端的步骤
  • 大多数的步骤都可以有多个数据跳，一个步骤的发送可以被设置为分发的复制，分发是目标步骤轮流接收记录，复制是所有记录被同时发送到所有的目标步骤。

2.4 Hop 跳

  • 跳就是步骤之间带箭头的连线，跳定义了步骤之间的数据通路
  • 跳实际上就是两个步骤之间的被称之为行集的数据行缓存（行集的大小可以在转换的设置里定义）
  • 当行集满了，向行集写数据的步骤将停止写入，直到行集里又有了空间
